Déploiement VM

Choix techniques et justifications matérielles

1. Hyperviseur : VirtualBox

Nous avons choisi Oracle VM VirtualBox pour sa nature Open Source et sa portabilité.

Justification Type 2

Contrairement au "Bare Metal" (Type 1), VirtualBox s'installe sur Windows (hôte). Cela permet de conserver l'usage bureautique de la machine physique tout en isolant l'environnement de développement (Sandbox), garantissant qu'une erreur critique dans la VM n'impacte pas l'ordinateur de l'étudiant[cite: 768, 769].

2. OS : Ubuntu 24.04 LTS

Installation de la version Desktop (GNOME) plutôt que Server.

Choix Pédagogique

Ce choix nous permet de vérifier le rendu du site web (localhost) directement dans la VM via Firefox avant de configurer la redirection de ports externe. Cela facilite grandement le débogage immédiat[cite: 773, 774].

Mémoire : 8 Go

Allocation de 8 Go de RAM.

Dimensionnement (Hôte 32 Go)

Comme l'hôte possède 32 Go de RAM, dédier 8 Go ne représente que 25% des ressources totales. Cela évite le swap sur la machine physique tout en assurant la fluidité de WebStorm et Apache dans la VM[cite: 778, 779].

Stockage : 25 Go

Disque virtuel dynamique.

Optimisation

25 Go suffisent pour l'OS et les fichiers du site. L'allocation dynamique permet de ne pas gaspiller l'espace disque de l'hôte[cite: 781, 782].

Configuration Initiale

Clavier Français (AZERTY) pour éviter les erreurs de syntaxe dans les fichiers de config[cite: 834].
Partitionnement "Effacer le disque" pour une installation propre et sécurisée sur le volume virtuel vierge[cite: 849, 850].
Fuseau Horaire Europe/Paris, crucial pour la cohérence des logs système[cite: 859].
